PyPy

From Wikipedia, the free encyclopedia

Jump to: navigation, search

PyPy is a reimplementation of Python written in Python itself, flexible and easy to experiment with. By enabling Python developers to play with the Python implementation in Python itself (rather than in the more cryptic C implementation), PyPy makes it easy to identify areas where the Python implementation can be improved. It also allows a developer to experiment with multiple implementations of specific features. One of the goals of the project is to generate an optimized Python implementation from PyPy that will run faster than the current C implementation,[1] although this goal has not yet been reached.

Note that PyPy is mainly a research and development project and does not by itself focus on getting a production-ready Python implementation at the moment.

PyPy consists of two major components:

  • the "Standard Interpreter", which is an implementation of a Python interpreter in a restricted subset of Python
  • a tool chain that can analyse the above-mentioned subset of Python and translate it to a low level language such as C.

One of the main developers of PyPy is Armin Rigo, who developed Psyco, a specializing just-in-time compiler for Python that speeds up Python code by 4-100x with almost no user intervention.

IIf

From Wikipedia, the free encyclopedia

Jump to: navigation, search

In computing, IIf is a function in several editions of the Visual Basic programming language that returns one of its two parameters based on the evaluation of an expression. It is similar to a conditional statement.

[edit]

Syntax

The syntax of the IIf function is as follows:

IIf(expr, truepart, falsepart)

All three parameters are required:

  • expr is the expression that is to be evaluated.
  • truepart defines what the IIf function returns if the evaluation of expr returns true.
  • falsepart defines what the IIf function returns if the evaluation of expr returns false.

An equivalent operator, ?:, exists in the C, C++, and related languages. Some of the problems with the IIf function, as discussed later, do not exist in C and C++ because of the strong type checking nature of the languages.

[edit]

Examples

These examples evaluate mathematical expressions and return one of two strings depending on the outcome.

result = IIf(5 < 10, "Yes it is", "No it isn't")     ' Returns "Yes it is"

result = IIf(2 + 2 = 5, "Correct", "Wrong")          ' Returns "Wrong"
[edit]

Criticisms

IIf has been criticised [1] for being inefficient and for having potential side-effects that the programmer might not realise at first. These criticisms mainly focus around the fact that the truepart and falsepart have no data type and a data type is automatically chosen when the parameter is assigned to. This is generally less efficient than assigning to parameters that explicitly declare a data type.

Another potential problem with IIf is that the function evaluates both truepart and falsepart regardless of which one it actually returns. Consider the following example:

value = 10
result = IIf(value = 10, TrueFunction, FalseFunction)

Although TrueFunction is the function that should be called, IIf will cause both TrueFunction and FalseFunction to be executed.

[edit]

Alternatives to IIf

In Visual Basic, IIf is not the sole way to evaluate and perform actions based on whether an expression is true or false.

The following example uses IIf:

result = IIf(test Mod 4 = 0, DoSomething, DoSomethingElse)

It could also be written in the following way, using standard conditional statements:

If test Mod 4 = 0 Then
  result = DoSomething
Else
  result = DoSomethingElse
End If

The above example would also eliminate the problem of IIf evaluating both its truepart and falsepart parameters.
